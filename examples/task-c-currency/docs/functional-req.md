## Функциональные требования

### ФТ-01: Выбор исходной и целевой валюты из списка
**Описание:** Система должна предоставлять два выпадающих списка (селекта) для выбора исходной и целевой валюты. В каждом списке должны быть доступны валюты из перечня БТ-05.
**Входные данные:** Пользователь выбирает значение из выпадающего списка (например, USD, EUR).
**Ожидаемый результат:** Выбранная валюта отображается в соответствующем поле выбора. При изменении любой из валют система автоматически пересчитывает результат.
**Источник:** БТ-01, UC-01

### ФТ-02: Получение и отображение актуального курса обмена
**Описание:** При старте приложения система должна выполнять fetch-запрос к https://open.er-api.com/v6/latest/USD для получения актуальных курсов валют. Полученные курсы должны кешироваться в localStorage с меткой времени. В интерфейсе система должна отображать текущий используемый курс пары (например, курс EUR) или сообщение об устаревших данных.
**Входные данные:** Запрос к внешнему API (GET-запрос).
**Ожидаемый результат:** Курсы валют загружены, сохранены в кеше и используются для конвертации. При успешной загрузке в интерфейсе отображается актуальный курс; при недоступности API — используется кеш с пометкой даты.
**Источник:** БТ-02, UC-01, UC-03

### ФТ-03: Мгновенный расчёт конвертации при вводе суммы
**Описание:** При вводе числа в поле «Сумма» и при изменении выбранных валют система должна автоматически (без нажатия кнопки) вычислять результат конвертации на основе текущего курса и отображать его рядом с полем ввода.
**Входные данные:** Числовое значение суммы (целое или дробное, с разделителем точка или запятая).
**Ожидаемый результат:** В специально отведённом месте (например, отдельное поле или блок) отображается значение: «{сумма} {исходная валюта} = {результат} {целевая валюта}». Значение должно обновляться при любом изменении входных данных.
**Источник:** БТ-03, UC-01

### ФТ-04: Кнопка смены валют местами
**Описание:** Между полями выбора исходной и целевой валюты должна располагаться кнопка с символом «⇄». При её нажатии система должна менять местами выбранные валюты и автоматически пересчитывать результат конвертации для той же суммы.
**Входные данные:** Клик по кнопке «⇄».
**Ожидаемый результат:** Значения в выпадающих списках меняются местами. Результат конвертации пересчитывается с учётом новой пары (обратный курс). Запись об операции добавляется в историю.
**Источник:** БТ-04, UC-02

### ФТ-05: Поддержка минимум 10 популярных валют
**Описание:** Список доступных валют в выпадающих меню должен включать как минимум: USD, EUR, RUB, GBP, JPY, CNY, TRY, KZT, BYN, GEL. Дополнительные валюты могут быть добавлены при наличии в ответе API.
**Входные данные:** Запрос к API (получение всех валют, поддерживаемых API).
**Ожидаемый результат:** В выпадающих списках отображаются перечисленные валюты (и, возможно, другие). Если API не содержит какую-либо из обязательных валют, система должна отобразить ошибку и использовать кеш или сообщить о проблеме.
**Источник:** БТ-05, Features

### ФТ-06: Использование кешированных курсов при недоступности API
**Описание:** Если при попытке загрузки актуальных курсов возникает ошибка сети, таймаут или HTTP-код ошибки, система должна загрузить последние успешно полученные курсы из localStorage (с их меткой времени) и продолжить работу с ними. При этом в верхней части интерфейса должно отображаться сообщение: «Курс от ДД.ММ.ГГГГ. Актуальные данные временно недоступны». Если кеш отсутствует или его возраст превышает 1 час, система должна блокировать конвертацию и показать сообщение о недоступности сервиса.
**Входные данные:** Ошибка при fetch-запросе (сбой сети, таймаут, HTTP-статус не 200).
**Ожидаемый результат:** Конвертер работает на кешированных данных с уведомлением пользователя. История конвертаций не очищается. При отсутствии/устаревании кеша конвертер отключается.
**Источник:** БТ-06, UC-03

### ФТ-07: Обработка некорректного ввода суммы
**Описание:** Система должна проверять введённое значение в поле «Сумма». Некорректными считаются: пустая строка, ноль, отрицательное число, нечисловые символы (кроме точки/запятой как десятичного разделителя). При любом некорректном вводе система должна отображать сообщение об ошибке красным цветом и не показывать результат конвертации. Поле ввода при этом может быть выделено красной рамкой.
**Входные данные:** Любое значение в поле суммы, не соответствующее правилам.
**Ожидаемый результат:** Сообщение об ошибке (например, «Некорректная сумма») отображается красным цветом, блок с результатом конвертации не отображается или остаётся пустым. Запись в историю не добавляется.
**Источник:** БТ-07, UC-01 (альтернативный поток)

### ФТ-08: Доступ через веб-браузер без установки ПО
**Описание:** Система должна быть реализована как веб-приложение (HTML, CSS, JavaScript), которое работает в браузере без установки дополнительного программного обеспечения. Для использования достаточно открыть страницу в любом поддерживаемом браузере.
**Входные данные:** URL-адрес приложения, введённый в адресную строку браузера.
**Ожидаемый результат:** Веб-страница загружается и корректно отображается, все функции конвертера доступны.
**Источник:** БТ-08
