# Юз-кейсы

### UC-01: Выполнение арифметического вычисления

**Источник:** БТ-01, БТ-05
**Актор:** Пользователь
**Предусловие:** Пользователь открыл веб-страницу калькулятора "MathBox".

**Основной поток:**
1. Пользователь вводит первое число, нажимая на цифровые кнопки.
2. Пользователь нажимает кнопку с нужной арифметической операцией (+, −, ×, ÷).
3. Пользователь вводит второе число, нажимая на цифровые кнопки.
4. Пользователь нажимает кнопку «=».
5. Система вычисляет результат.
6. Система отображает результат в основном поле вывода.
7. Система добавляет выполненную операцию в историю вычислений (до 10 последних).
8. Система сохраняет обновлённую историю в localStorage.

**Альтернативные потоки:**
- В шаге 5, если второе число равно нулю и выбрана операция деления (÷): система прерывает вычисление, отображает в основном поле сообщение «Ошибка: деление на ноль» с красной подсветкой и переходит к шагу 7 (операция с ошибкой добавляется в историю).

**Постусловие:** Результат вычисления отображён на экране. Операция занесена в историю.

### UC-02: Обработка ошибки "Деление на ноль"

**Источник:** БТ-04
**Актор:** Пользователь
**Предусловие:** Пользователь ввёл первое число и выбрал операцию деления (÷).

**Основной поток:**
1. Пользователь вводит «0» в качестве второго числа.
2. Пользователь нажимает кнопку «=».
3. Система определяет попытку деления на ноль.
4. Система прерывает стандартное вычисление.
5. Система отображает в основном поле вывода сообщение «Ошибка: деление на ноль» с красной подсветкой.
6. Система добавляет запись о попытке деления на ноль в историю вычислений.

**Альтернативные потоки:**
- В любой момент после появления ошибки пользователь может нажать любую цифровую кнопку или кнопку операции: сообщение об ошибке очищается, начинается ввод нового выражения.

**Постусловие:** В поле вывода отображается сообщение об ошибке. Запись о неудачной операции добавлена в историю.

### UC-03: Сброс текущего ввода и калькулятора

**Источник:** БТ-03
**Актор:** Пользователь
**Предусловие:** Калькулятор находится в любом состоянии (начальный экран, частичный ввод, отображение результата или ошибки).

**Основной поток:**
1. Пользователь нажимает кнопку «C».
2. Система полностью очищает основное поле ввода/вывода.
3. Система сбрасывает все промежуточные состояния (забывает введённые числа и операцию).

**Альтернативные потоки:**
- Нет.

**Постусловие:** Калькулятор возвращён в начальное, очищенное состояние, готов к новому вводу.

### UC-04: Просмотр истории вычислений

**Источник:** БТ-02
**Актор:** Пользователь
**Предусловие:** Пользователь открыл веб-страницу калькулятора "MathBox". В localStorage или текущей сессии есть записи о предыдущих вычислениях.

**Основной поток:**
1. Система автоматически загружает историю последних до 10 операций из localStorage при загрузке страницы.
2. Система отображает историю в боковой панели интерфейса в формате (например): «5 + 3 = 8».
3. Пользователь просматривает список операций в истории.

**Альтернативные потоки:**
- Если история пуста: боковая панель истории остаётся пустой или отображает заглушку (например, «История операций пуста»).

**Постусловие:** Пользователь видит список последних операций. История сохранена между сессиями.

### UC-05: Копирование результата в буфер обмена

**Источник:** Характеристики
**Актор:** Пользователь
**Предусловие:** В основном поле вывода калькулятора отображается валидный результат вычисления (не сообщение об ошибке).

**Основной поток:**
1. Пользователь нажимает кнопку «Копировать», расположенную рядом с полем результата.
2. Система копирует числовое значение из поля вывода в буфер обмена устройства пользователя.
3. Система предоставляет визуальную обратную связь (например, временное изменение иконки кнопки или текст «Скопировано»).

**Альтернативные потоки:**
- Если в поле вывода отображается сообщение об ошибке или оно пустое: кнопка «Копировать» неактивна (затенена).

**Постусловие:** Числовой результат последней успешной операции находится в буфере обмена пользователя.
