# Юз-кейсы

### UC-01: Создание проекта

**Источник:** БТ-01  
**Актор:** Пользователь  
**Предусловие:** Пользователь открыл приложение, видит боковую панель.

**Основной поток:**
1. Пользователь нажимает кнопку «+ Проект» на боковой панели.
2. Система открывает диалог создания проекта с полем для названия и палитрой цветов (6–8 цветов).
3. Пользователь вводит название проекта и выбирает цветовую метку.
4. Пользователь нажимает «Создать».
5. Система создаёт новый проект с четырьмя пустыми колонками (Бэклог, В работе, На проверке, Готово), сохраняет его в localStorage и переходит на доску проекта.
6. Проект появляется в списке проектов на боковой панели с цветной точкой.

**Альтернативные потоки:**
- Пустое название: Пользователь не вводит название и нажимает «Создать». Система показывает ошибку валидации, проект не создаётся.

**Постусловие:** Проект создан и отображается в системе, все данные сохранены.

### UC-02: Создание задачи

**Источник:** БТ-03  
**Актор:** Пользователь  
**Предусловие:** Пользователь находится на доске проекта с хотя бы одной колонкой.

**Основной поток:**
1. Пользователь нажимает «+ Задача» в шапке доски (задача помещается в колонку «Бэклог») или кнопку «+ Задача» внутри конкретной колонки.
2. Система открывает модальное окно с формой: название (обязательно), описание (необязательно), приоритет (выбор: высокий/средний/низкий, по умолчанию средний), дедлайн (date picker, необязательно), теги (ввод через запятую).
3. Пользователь заполняет обязательное поле названия и при необходимости другие поля.
4. Пользователь нажимает «Создать».
5. Система добавляет карточку задачи в целевую колонку, обновляет счётчик задач в шапке колонки и сохраняет данные в localStorage.

**Альтернативные потоки:**
- Пустое название задачи: Пользователь нажимает «Создать», не указав название. Система подсвечивает поле названия красным, задача не создаётся.

**Постусловие:** Задача создана и отображается в колонке, счётчик увеличен.

### UC-03: Перемещение задачи между колонками

**Источник:** БТ-04  
**Актор:** Пользователь  
**Предусловие:** На доске проекта есть хотя бы одна задача, которая находится не в крайней колонке (для перемещения вперёд) или не в первой колонке (для перемещения назад).

**Основной поток:**
1. Пользователь видит карточку задачи с кнопками «→» (вперёд) и «←» (назад).
2. Пользователь нажимает кнопку «→».
3. Система перемещает задачу в следующую колонку по порядку (Бэклог → В работе → На проверке → Готово).
4. Счётчики задач в исходной и целевой колонках обновляются.
5. Изменение сохраняется в localStorage.

**Альтернативные потоки:**
- Задача в последней колонке («Готово»): Кнопка «→» неактивна или скрыта. Пользователь может нажать только «←».
- Перемещение назад: Пользователь нажимает «←». Система перемещает задачу в предыдущую колонку (Готово → На проверке → В работе → Бэклог). Если задача в первой колонке («Бэклог»), кнопка «←» неактивна.

**Постусловие:** Задача перемещена в соответствующую колонку, счётчики обновлены.

### UC-04: Редактирование задачи

**Источник:** БТ-07  
**Актор:** Пользователь  
**Предусловие:** На доске отображается хотя бы одна задача.

**Основной поток:**
1. Пользователь нажимает кнопку «✎» на карточке задачи или кликает по самой карточке.
2. Система открывает модальное окно, предварительно заполненное текущими данными задачи (название, описание, приоритет, дедлайн, теги).
3. Пользователь вносит изменения в любые поля.
4. Пользователь нажимает «Сохранить».
5. Система обновляет карточку задачи на доске и сохраняет изменения в localStorage.

**Альтернативные потоки:** Отсутствуют (в БП не описаны другие варианты).

**Постусловие:** Задача изменена, данные обновлены.

### UC-05: Удаление задачи

**Источник:** БТ-07  
**Актор:** Пользователь  
**Предусловие:** На доске отображается хотя бы одна задача.

**Основной поток:**
1. Пользователь нажимает кнопку «✕» на карточке задачи.
2. Система показывает подтверждение: «Удалить задачу "<название>"?».
3. Пользователь подтверждает удаление.
4. Система удаляет задачу из колонки, обновляет счётчик колонки и сохраняет изменения в localStorage.

**Альтернативные потоки:** Отсутствуют (в БП не описаны другие варианты).

**Постусловие:** Задача удалена, счётчик уменьшен.

### UC-06: Фильтрация и поиск задач

**Источник:** БТ-09, БТ-10  
**Актор:** Пользователь  
**Предусловие:** Пользователь находится на доске проекта (в любом представлении) и в проекте есть хотя бы одна задача.

**Основной поток:**
1. Пользователь вводит текст в поле поиска.
2. Система немедленно скрывает карточки задач, в названии или описании которых нет введённого текста (регистронезависимый поиск).
3. Пользователь выбирает фильтр по приоритету из выпадающего списка (Все / Высокий / Средний / Низкий).
4. Пользователь выбирает фильтр по тегу из выпадающего списка всех тегов, встречающихся в задачах проекта.
5. Система применяет все активные фильтры с AND-логикой: отображаются только задачи, удовлетворяющие всем условиям.
6. На кнопке фильтра отображается счётчик активных фильтров. Если хотя бы один фильтр активен, появляется кнопка «Сбросить фильтры».

**Альтернативные потоки:** Отсутствуют (в БП не описаны отдельные альтернативы; сброс фильтров — часть основного потока).

**Постусловие:** Отображаются только задачи, соответствующие критериям поиска и фильтрации.

### UC-07: Переключение представлений доски

**Источник:** БТ-08  
**Актор:** Пользователь  
**Предусловие:** Пользователь находится на доске проекта (в режиме канбан-доски или списка).

**Основной поток:**
1. Пользователь нажимает кнопку «Список» в шапке доски (если текущее представление — канбан) или кнопку «Доска» (если текущее представление — список).
2. Система переключает вид:
   - При переходе к списку: отображается таблица с колонками «Название», «Приоритет», «Статус», «Дедлайн», «Теги». Строки можно сортировать по дедлайну и приоритету.
   - При переходе к доске: отображается канбан-доска с колонками.
3. Фильтры и поиск, если были активны, сохраняются и применяются в новом представлении.

**Альтернативные потоки:** Отсутствуют.

**Постусловие:** Представление изменено, задачи отображаются в выбранном режиме.

### UC-08: Просмотр дашборда

**Источник:** БТ-11  
**Актор:** Пользователь  
**Предусловие:** В системе есть хотя бы один проект и хотя бы одна задача.

**Основной поток:**
1. Пользователь нажимает пункт «Дашборд» в навигации (боковой панели).
2. Система отображает:
   - Карточки-метрики: общее число задач, количество задач в статусах «В работе», «На проверке», «Готово», число просроченных задач.
   - Для каждого проекта — блок с названием, полосой прогресса (процент задач в «Готово») и разбивкой задач по статусам.
   - Список «Ближайшие дедлайны»: задачи с дедлайном в ближайшие 7 дней (название, проект, дата), отсортированные по дате.
3. Все данные обновляются в реальном времени при изменениях.

**Альтернативные потоки:** Отсутствуют.

**Постусловие:** Дашборд отображает актуальную статистику по всем проектам.
