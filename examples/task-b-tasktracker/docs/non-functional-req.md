### Документ 1: non-functional-req.md

### НФТ-01: Время отклика при фильтрации и поиске
**Категория:** производительность
**Описание:** При вводе текста в поле поиска или выборе фильтра по приоритету/тегу, интерфейс должен скрывать/отображать задачи немедленно, без задержки более 200 мс. Фильтрация должна выполняться на стороне клиента без отправки запросов на сервер.
**Источник:** Features (живая фильтрация), БТ-09, БТ-10

### НФТ-02: Работа без дополнительного ПО
**Категория:** совместимость
**Описание:** Приложение должно корректно работать как одностраничное приложение (SPA) в последних версиях браузеров Chromium, Firefox, Safari и Edge без установки каких-либо плагинов, расширений или серверного ПО. Допускается использование только HTML, CSS и нативного JavaScript.
**Источник:** БТ-15, Features

### НФТ-03: Корректная работа на разных разрешениях экрана
**Категория:** удобство использования
**Описание:** Интерфейс должен корректно отображаться на экранах с разрешением от 1024x768 пикселей и выше. Боковая панель (sidebar) должна быть фиксированной ширины ~220px. Канбан-доска должна поддерживать горизонтальный скролл при недостатке места для отображения четырёх колонок.
**Источник:** Features

### НФТ-04: Сохранение данных при обновлении страницы
**Категория:** надёжность
**Описание:** Все данные о проектах, задачах и их статусе должны автоматически сохраняться в localStorage при каждом изменении. При перезагрузке страницы или случайном закрытии браузера данные должны полностью восстанавливаться без потерь.
**Источник:** БТ-12

### НФТ-05: Защита от потери данных при подтверждении удаления
**Категория:** безопасность
**Описание:** При попытке удаления задачи система должна запрашивать подтверждение от пользователя в виде диалогового окна. Без явного подтверждения задача не должна удаляться. Удаление проекта или задачи должно быть необратимым только после подтверждения.
**Источник:** UC-05, БТ-07

### НФТ-06: Белая тема по умолчанию
**Категория:** удобство использования
**Описание:** Приложение должно использовать светлую (белую) тему оформления по умолчанию при первой загрузке. Цвет фона основных областей (канбан-доски, боковой панели, модальных окон) должен быть белым, текст — тёмным для обеспечения читаемости.
**Источник:** Инструкция по изменению
