# Бизнес-процесс (БП) — Управление задачами в TaskFlow

**Актор:** Пользователь

---

## БП-1: Создание проекта

1. Пользователь открывает приложение.
2. Пользователь нажимает кнопку «+ Новый проект».
3. Пользователь вводит название проекта и выбирает цветовую метку из палитры (6–8 цветов).
4. Система создаёт проект с четырьмя пустыми колонками и переходит на его доску.
5. Система сохраняет проект в localStorage.

### Альтернативный поток — пустое название
3а. Пользователь оставляет поле пустым и нажимает «Создать».
3б. Система показывает ошибку валидации, проект не создаётся.

---

## БП-2: Создание задачи

1. Пользователь находится на доске проекта.
2. Пользователь нажимает «+ Задача» в любой колонке (задача попадает именно в эту колонку) или «+ Новая задача» в шапке (задача попадает в «Бэклог»).
3. Открывается модальное окно с формой: название (обязательно), описание (необязательно), приоритет (выбор из 3 вариантов, по умолчанию «средний»), дедлайн (date picker, необязательно), теги (ввод через запятую).
4. Пользователь заполняет форму и нажимает «Создать».
5. Система добавляет карточку задачи в нужную колонку, обновляет счётчик, сохраняет в localStorage.

### Альтернативный поток — пустое название задачи
4а. Пользователь нажимает «Создать» без названия.
4б. Система подсвечивает поле красным, задача не создаётся.

---

## БП-3: Перемещение задачи между колонками

1. Пользователь видит карточку задачи в колонке.
2. Пользователь нажимает кнопку «→» на карточке задачи.
3. Система перемещает задачу в следующую колонку (Бэклог → В работе → На проверке → Готово).
4. Система обновляет счётчики обеих колонок и сохраняет изменение.

### Альтернативный поток — задача уже в последней колонке
2а. Пользователь нажимает «→» на задаче в колонке «Готово».
2б. Кнопка «→» неактивна или скрыта.

### Альтернативный поток — перемещение назад
2а. Пользователь нажимает кнопку «←» на карточке задачи.
2б. Система перемещает задачу в предыдущую колонку.

---

## БП-4: Редактирование задачи

1. Пользователь нажимает на карточку задачи (или кнопку «✎»).
2. Открывается то же модальное окно, заполненное текущими данными.
3. Пользователь вносит изменения и нажимает «Сохранить».
4. Система обновляет карточку и localStorage.

---

## БП-5: Удаление задачи

1. Пользователь нажимает кнопку «✕» на карточке задачи.
2. Система показывает подтверждение: «Удалить задачу "<название>"?».
3. Пользователь подтверждает.
4. Система удаляет задачу, обновляет счётчик колонки.

---

## БП-6: Фильтрация и поиск

1. Пользователь вводит текст в поле поиска — карточки, не содержащие текст в названии или описании, скрываются немедленно.
2. Пользователь выбирает фильтр по приоритету (Все / Высокий / Средний / Низкий) — карточки фильтруются.
3. Пользователь выбирает фильтр по тегу из выпадающего списка всех тегов проекта.
4. Фильтры суммируются (AND-логика): показываются только задачи, удовлетворяющие всем активным фильтрам.
5. Счётчик активных фильтров отображается на кнопке «Фильтры».

---

## БП-7: Переключение представлений

1. Пользователь нажимает кнопку «Список» в шапке доски.
2. Система переключает вид с канбан-доски на табличный список: строки с колонками «Название», «Приоритет», «Статус», «Дедлайн», «Теги».
3. Пользователь нажимает «Доска» — вид возвращается к канбан-доске.

---

## БП-8: Просмотр дашборда

1. Пользователь нажимает «Дашборд» в навигации.
2. Система отображает:
   - Суммарную статистику по всем проектам: всего задач, в работе, на проверке, готово, просрочено.
   - Для каждого проекта — полоса прогресса (% задач в «Готово»).
   - Список ближайших дедлайнов (следующие 7 дней) по всем проектам.
