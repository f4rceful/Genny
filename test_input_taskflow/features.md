# Характеристики (Features) — TaskFlow

- Название приложения — «TaskFlow»
- Тёмная тема по умолчанию (фон #0f1117, карточки #1c1f2e, акцент #6366f1 — индиго)
- Все надписи на русском языке
- Одностраничное приложение (SPA) — только HTML + CSS + JS, без фреймворков и сборщиков

## Навигация

- Боковая панель (sidebar) шириной ~220px: список проектов с цветными точками, кнопка «+ Проект», пункт «Дашборд»
- Активный проект выделяется в sidebar
- Архивированные проекты скрыты, но доступны через «Показать архив» внизу sidebar

## Канбан-доска

- 4 колонки одинаковой ширины, горизонтальный скролл при необходимости
- Шапка колонки: название + счётчик задач + кнопка «+ Задача»
- Карточка задачи содержит:
  - Цветную полоску приоритета слева (красный — высокий, жёлтый — средний, серый — низкий)
  - Название задачи (жирный)
  - Описание (до 2 строк, обрезается с «…»)
  - Теги в виде цветных пилюль (цвет тега генерируется из хеша его имени)
  - Дата дедлайна с иконкой (красная если просрочено, жёлтая если сегодня, белая иначе)
  - Кнопки действий: «←» (назад), «✎» (редактировать), «✕» (удалить), «→» (вперёд)
- Кнопки «←» и «→» скрыты если перемещение в соответствующую сторону невозможно

## Список (табличный вид)

- Таблица с колонками: приоритет (цветная точка), название, статус, дедлайн, теги, действия
- Строки можно сортировать по дедлайну и приоритету (клик по заголовку колонки)
- Чётные строки имеют слегка другой фон для читаемости

## Поиск и фильтры

- Поле поиска в шапке доски — живая фильтрация без кнопки
- Выпадающий список приоритетов
- Выпадающий список тегов (собирается из всех задач проекта)
- Кнопка «Сбросить фильтры» появляется когда активен хотя бы один фильтр

## Модальное окно задачи

- Открывается поверх доски с затемнённым фоном
- Закрывается по клику вне окна или по кнопке «✕»
- Форма: поле названия, textarea описания, select приоритета, input[type=date] дедлайна, input тегов (подсказка: «через запятую»)
- Кнопки «Отмена» и «Создать» / «Сохранить»

## Дашборд

- Карточки-метрики в одну строку: «Всего задач», «В работе», «На проверке», «Готово», «Просрочено»
- Для каждого проекта — блок с названием, полосой прогресса и числом задач по статусам
- Блок «Ближайшие дедлайны» — список задач с дедлайном в течение 7 дней (название, проект, дата), отсортированный по дате

## Хранение данных

- Все данные хранятся в localStorage под ключом `taskflow_data`
- Структура: `{ projects: [...], tasks: [...] }`
- Данные автоматически сохраняются при каждом изменении
- При первом запуске создаётся демо-проект «Мой первый проект» с 3–4 примерными задачами в разных колонках, чтобы интерфейс не выглядел пустым
