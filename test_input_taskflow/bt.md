# Бизнес-требования (БТ) — Трекер задач TaskFlow

| ID     | Требование                                                                                                                                    | Обязательность  |
|--------|-----------------------------------------------------------------------------------------------------------------------------------------------|-----------------|
| БТ-01  | Пользователь должен иметь возможность создавать несколько проектов, каждый с названием и цветовой меткой                                      | Обязательное    |
| БТ-02  | Внутри каждого проекта задачи должны отображаться в виде канбан-доски с колонками: «Бэклог», «В работе», «На проверке», «Готово»              | Обязательное    |
| БТ-03  | Пользователь должен иметь возможность создавать задачи с полями: название, описание, приоритет (высокий/средний/низкий), дедлайн, теги        | Обязательное    |
| БТ-04  | Пользователь должен иметь возможность перемещать задачу между колонками нажатием кнопки «→» / «←» (drag-and-drop необязателен)              | Обязательное    |
| БТ-05  | Задачи с истёкшим дедлайном должны визуально выделяться (красная метка «Просрочено»)                                                         | Обязательное    |
| БТ-06  | Задачи с дедлайном сегодня должны визуально выделяться (жёлтая метка «Сегодня»)                                                              | Обязательное    |
| БТ-07  | Пользователь должен иметь возможность редактировать и удалять задачи                                                                         | Обязательное    |
| БТ-08  | Пользователь должен иметь возможность переключаться между режимом канбан-доски и списком (табличное представление)                            | Обязательное    |
| БТ-09  | Пользователь должен иметь возможность фильтровать задачи по приоритету, тегу и статусу колонки                                               | Обязательное    |
| БТ-10  | Пользователь должен иметь возможность искать задачи по названию и описанию в реальном времени                                                 | Обязательное    |
| БТ-11  | На главном дашборде должна отображаться статистика: общее число задач, распределение по статусам, число просроченных, прогресс по проекту в % | Обязательное    |
| БТ-12  | Данные всех проектов и задач должны сохраняться в localStorage и восстанавливаться при следующем открытии страницы                            | Обязательное    |
| БТ-13  | Пользователь должен иметь возможность архивировать завершённый проект (скрыть из списка, но не удалять данные)                                | Желательное     |
| БТ-14  | Счётчики задач на каждой колонке должны обновляться в реальном времени                                                                       | Обязательное    |
| БТ-15  | Интерфейс должен корректно работать в браузере без установки дополнительного ПО                                                               | Обязательное    |
