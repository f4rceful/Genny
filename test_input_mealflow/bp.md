# Бизнес-процесс (БП) — MealFlow

**Актор:** Пользователь

---

## БП-1: Создание рецепта

1. Пользователь нажимает «+ Рецепт» в разделе «Рецепты».
2. Открывается модальное окно с формой: название, описание, кухня (select), сложность (Лёгкий / Средний / Сложный), время (мин), порций, теги (через запятую), URL фото (необязательно).
3. В блоке «Ингредиенты» пользователь добавляет позиции: вводит название ингредиента → система предлагает автодополнение из базы ингредиентов → пользователь выбирает или добавляет новый → вводит количество и единицу измерения.
4. Пользователь нажимает «Сохранить». Система валидирует: название не пустое, хотя бы 1 ингредиент, количество > 0.
5. Рецепт сохраняется, пользователь видит его карточку в галерее.

### Альтернативный поток — новый ингредиент не в базе
3а. Пользователь вводит название ингредиента, которого нет в базе.
3б. Появляется кнопка «Добавить новый ингредиент».
3в. Открывается мини-форма: название, калории/100г, белки/100г, жиры/100г, углеводы/100г, категория (select).
3г. Ингредиент добавляется в базу и сразу используется в рецепте.

---

## БП-2: Добавление рецепта в план питания

1. Пользователь переходит в раздел «План питания» — отображается недельная сетка (7 колонок × 4 строки: завтрак, обед, ужин, перекус).
2. Пользователь нажимает «+» в нужной ячейке (день × приём пищи).
3. Открывается панель поиска рецептов с фильтрами (поиск по названию, фильтр по кухне).
4. Пользователь выбирает рецепт, указывает количество порций (по умолчанию — то, на сколько рассчитан рецепт).
5. Система добавляет блок рецепта в ячейку с указанием порций и краткими КБЖУ.
6. Сумма КБЖУ дня пересчитывается и отображается в заголовке колонки.

### Альтернативный поток — ячейка уже занята другим рецептом
4а. В одной ячейке может быть несколько рецептов. Новый добавляется под предыдущим.

---

## БП-3: Удаление рецепта из плана

1. Пользователь видит блок рецепта в ячейке плана.
2. Пользователь нажимает «✕» на блоке.
3. Система удаляет запись из плана, пересчитывает КБЖУ дня.

---

## БП-4: Масштабирование порций в карточке рецепта

1. Пользователь открывает карточку рецепта (клик на карточку в галерее).
2. В детальном виде отображаются ингредиенты и шаги. Есть спиннер «Порций: [N]».
3. Пользователь меняет значение спиннера.
4. Все количества ингредиентов и КБЖУ на порцию пересчитываются мгновенно.
5. Изменение не сохраняется в базу — это только локальный просмотр.

---

## БП-5: Генерация и работа со списком покупок

1. Пользователь переходит в раздел «Список покупок».
2. Система автоматически агрегирует ингредиенты из всего недельного плана:
   - Одинаковые ингредиенты суммируются.
   - Конвертация: если один рецепт требует 200 г муки, другой — 0.3 кг, итого 0.5 кг.
   - Если единица не конвертируема (шт + г) — показываются отдельными строками.
3. Список отображается сгруппированным по категориям: каждая группа раскрывается/сворачивается.
4. Пользователь нажимает на позицию — она отмечается как купленная (зачёркивается, перемещается в конец группы).
5. Кнопка «Сбросить все» снимает все отметки.
6. Кнопка «Очистить купленные» удаляет купленные позиции из отображения.

### Альтернативный поток — план пустой
2а. Список покупок пуст, отображается подсказка «Добавьте рецепты в план питания».

---

## БП-6: Просмотр анализа питания

1. Пользователь переходит в раздел «Анализа».
2. Система отображает:
   - Карточки-метрики недели: суммарные калории, белки, жиры, углеводы.
   - Столбчатая диаграмма (SVG): калории по дням пн–вс, с горизонтальной линией нормы (2000 ккал).
   - Круговая диаграмма (SVG): распределение БЖУ в граммах за неделю.
   - Таблица по дням: для каждого дня — список приёмов пищи с суммой КБЖУ.
3. Если день пустой — в диаграмме показывается 0, строка в таблице серая.

---

## БП-7: Фильтрация и поиск рецептов

1. Пользователь находится в разделе «Рецепты» — галерея карточек.
2. Панель фильтров содержит: строка поиска (название / ингредиент), select кухни, select тега, ползунок макс. времени (5–180 мин), ползунок макс. калорий/порцию (100–1500 ккал), чекбокс «Только избранные».
3. Все фильтры работают одновременно (AND-логика), карточки фильтруются мгновенно.
4. Количество найденных рецептов отображается над галереей.
5. Кнопка «Сбросить» сбрасывает все фильтры.

---

## БП-8: Редактирование и удаление рецепта

1. Пользователь открывает карточку рецепта, нажимает «✎ Редактировать».
2. Открывается та же форма, заполненная текущими данными.
3. Пользователь вносит изменения, нажимает «Сохранить».
4. Если рецепт используется в плане, КБЖУ плана пересчитываются.

### Удаление рецепта
1. Пользователь нажимает «🗑 Удалить» на карточке.
2. Система показывает предупреждение: «Рецепт "<название>" также будет удалён из плана питания. Продолжить?»
3. После подтверждения рецепт удаляется из базы и из плана.
