# Бизнес-требования (БТ) — Планировщик питания MealFlow

| ID     | Требование                                                                                                                                                          | Обязательность |
|--------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------|
| БТ-01  | Пользователь должен иметь возможность создавать рецепты с полями: название, описание, кухня, сложность, время приготовления (мин), кол-во порций, теги, URL фото    | Обязательное   |
| БТ-02  | Каждый рецепт содержит список ингредиентов: название, количество, единица измерения (г, кг, мл, л, шт, ст.л., ч.л., щепотка)                                       | Обязательное   |
| БТ-03  | Система должна вести базу ингредиентов с питательной ценностью на 100 г: калории, белки, жиры, углеводы                                                             | Обязательное   |
| БТ-04  | Пользователь должен иметь возможность добавить рецепт в план питания на конкретный день (пн–вс) и приём пищи (завтрак / обед / ужин / перекус)                     | Обязательное   |
| БТ-05  | При добавлении рецепта в план пользователь указывает количество порций — система пересчитывает все ингредиенты пропорционально                                       | Обязательное   |
| БТ-06  | Система автоматически генерирует список покупок из всего недельного плана: одинаковые ингредиенты суммируются с конвертацией единиц (г + кг = кг, мл + л = л)      | Обязательное   |
| БТ-07  | Список покупок группируется по категориям ингредиентов (молочное, мясо, овощи, крупы, специи и др.)                                                                 | Обязательное   |
| БТ-08  | Пользователь может отмечать купленные позиции в списке покупок галочкой; купленные отображаются зачёркнутыми                                                        | Обязательное   |
| БТ-09  | Система рассчитывает КБЖУ (калории, белки, жиры, углеводы) для каждого дня и для всей недели на основании ингредиентов и порций                                    | Обязательное   |
| БТ-10  | На экране анализа питания отображается столбчатая диаграмма калорий по дням (CSS/SVG, без сторонних библиотек) и круговая диаграмма распределения БЖУ за неделю    | Обязательное   |
| БТ-11  | Пользователь должен иметь возможность фильтровать рецепты по: кухне, тегу, максимальному времени приготовления, максимальной калорийности на порцию                | Обязательное   |
| БТ-12  | Пользователь может отметить рецепт как «избранное» и быстро фильтровать только избранные                                                                           | Обязательное   |
| БТ-13  | Поиск рецептов по названию и ингредиентам в реальном времени                                                                                                       | Обязательное   |
| БТ-14  | Пользователь может масштабировать порции прямо в карточке рецепта — все граммовки пересчитываются без сохранения изменений в оригинал                              | Обязательное   |
| БТ-15  | Пользователь может удалить рецепт из плана конкретного дня, не удаляя сам рецепт из базы                                                                           | Обязательное   |
| БТ-16  | Все данные (рецепты, план, список покупок) хранятся в localStorage и восстанавливаются при перезагрузке                                                             | Обязательное   |
| БТ-17  | При первом запуске загружается демо-набор: минимум 6 рецептов из разных кухонь, частично заполненный недельный план и предзаполненная база ингредиентов (≥20 позиций) | Обязательное   |
